    Section 2.6 LU-decomposition

    In this section, we see some issues that can arise when computers use the Gauss-Jordan Elimination algorithm, and we learn some of what computers do to compute solutions more efficiently, especially when we want to solve equations \(AX = B\) for several different column matrices \(B\text{.}\)
    ðŸ”—

    Subsection Prepare

    Computers are very fast at performing computations, but sometimes our brains are still better. For instance, we know that \(0.1 + 0.2 = 0.3\text{,}\) but execute the code below to see what Python says.
    ðŸ”—
    Running the code above shows that when using Python, the computer does not have the same answer for 0.1 + 0.2 and 0.3. Computers perform arithmetic using base 2 numbers, which means that numbers we enter in decimal form, such as \(0.1\text{,}\) must be converted to base 2. Even though 0.1 has a simple decimal (base 10) form, its representation in base 2 is the repeating decimal
    \begin{equation*} 0.000110011001100110011001100110011001100110011\ldots\text{.}\text{,} \end{equation*}
    To accurately represent this number inside a computer would require infinitely many digits, but a computer can only hold a finite number of digits. Thus, computers are necessarily using an approximation when internally storing most numbers.
    ðŸ”—
    To see this phenomenon in another way, modify and run the code above to display 0.3 itself to 15 decimal places, and then to 25 decimal places.
    ðŸ”—
    Because computers use a finite number of digits to approximate numbers, they are also prone to round off errors. Gauss-Jordan elimination, when applied to an \(n\times n\) matrix, requires approximately \(\frac 23 n^3\) operations of multiplying and adding numbers. The examples we have seen are small enough to do by hand, but applications in computer graphics, machine learning, and many others can easily have hundreds or thousands of variables. If we have a \(1000\times1000\) matrix, performing the Gauss-Jordan algorithm would take roughly a billion operations, and any errors introduced in an operation early on could accumulate and compound as we use that erroneous number in the next operation, and the next, and the next.
    ðŸ”—
    There are a few things computers do to mitigate the issue of numerical errors. You may have noticed that when we perform Gauss-Jordan elimination by hand, we primarily use only two of the three row operations: scaling and multiply-and-add. We swap rows only during the forward steps and only if there was a \(0\) in a position we needed to use to eliminate nonzero numbers still appearing in the same column.
    ðŸ”—
    However, when computers perform the forward steps, they swap rows so that they are using the number with the largest absolute value to eliminate the other entries in the same column. This is called partial pivoting, and it helps to reduce the number and magnitude of round off errors.
    ðŸ”—
    Another thing computers do to mitigate operational errors is rewrite the coefficient matrix as a product of matrices which are easier to work with, not unlike in algebra when we solve a quadratic equation by factoring it. We are going to use triangular matrices in this section, but there are many other ways to factor, or decompose, a matrix and each factorization is useful in various different applications and objectives.
    ðŸ”—

    Example 2.6.3. Solving a system with an LU factorization.

    Suppose we are trying to solve the system
    \begin{align*} x_1 + 2x_2 \amp = -1\\ 3x_1 + 5x_2 \amp = -2\text{,} \end{align*}
    which we can view as
    \begin{equation*} AX=B\text{,} \end{equation*}
    for \(A=\begin{bmatrix}1 \amp 2 \\ 3 \amp 5 \end{bmatrix}\text{,}\) \(X=\begin{bmatrix}x_1 \\ x_2 \end{bmatrix}\text{,}\) and \(B=\begin{bmatrix}-1 \\ -2 \end{bmatrix}\text{.}\)
    ðŸ”—
    We know we could do this by augmenting and row reducing, or by calculating the inverse of the coefficient matrix. Letâ€™s see yet another way to solve this system.
    ðŸ”—
    Take the coefficient matrix \(A=\begin{bmatrix} 1 \amp 2 \\ 3 \amp 5 \end{bmatrix}\) and use an elementary matrix to perform the first step in Gauss-Jordan elimination, which is \(-3R_1+R_2 \rightarrow R_2\text{.}\) Then \(E_1=\begin{bmatrix} 1 \amp 0 \\ -3 \amp 1 \end{bmatrix}\) and \(E_1A\) is
    \begin{align*} E_1A\amp =\begin{bmatrix} 1 \amp 0 \\ -3 \amp 1 \end{bmatrix}\begin{bmatrix} 1 \amp 2 \\ 3 \amp 5 \end{bmatrix}\\ \amp = \begin{bmatrix} 1 \amp 2 \\ 0 \amp -1 \end{bmatrix} \text{.} \end{align*}
    Notice that \(E_1\) is lower-triangular and \(E_1A\) is upper-triangular. Define \(U=E_1A=\begin{bmatrix} 1 \amp 2 \\ 0 \amp -1 \end{bmatrix}\text{.}\)
    ðŸ”—
    In our work in SectionÂ 2.5 as a whole and in ActivityÂ 2.5.5 in particular, we saw that every elementary matrix is invertible. In fact, \(E_1^{-1}=\begin{bmatrix} 1 \amp 0 \\ 3 \amp 1 \end{bmatrix}\text{,}\) which is also lower-triangular. Define \(L=E_1^{-1}\text{.}\)
    ðŸ”—
    Then
    \begin{align*} A \amp =\big(E_1^{-1}E_1\big)A = \begin{bmatrix} 1 \amp 0 \\ 3 \amp 1 \end{bmatrix} \begin{bmatrix} 1 \amp 0 \\ -3 \amp 1 \end{bmatrix}\begin{bmatrix} 1 \amp 2 \\ 3 \amp 5 \end{bmatrix}\\ \amp =E_1^{-1}\big(E_1 A\big) = \begin{bmatrix} 1 \amp 0 \\ 3 \amp 1 \end{bmatrix} \begin{bmatrix} 1 \amp 2 \\ 0 \amp -1 \end{bmatrix}\\ \amp =LU \text{,} \end{align*}
    a lower-triangular matrix multiplied by an upper-triangular matrix.
    ðŸ”—
    How has this helped us? A linear system with a triangular coefficient matrix does not need to be row reduced any further. It is more efficient to solve such a system with substitution, coming full-circle to what we did back in SectionÂ 1.1 before we ever learned about elimination and row operations.
    ðŸ”—
    We are trying to solve the linear system \(AX=B\text{,}\) and we have seen how to rewrite \(A\) as \(LU\text{,}\) so we are now trying to solve \(LUX=B\text{.}\) We know that \(U\) is a \(2\times 2\) matrix and \(X\) is a \(2\times 1\) matrix. That means that \(UX\) is also a \(2\times 1\) matrix. For the moment, letâ€™s define that \(Y=UX=\begin{bmatrix} y_1 \\ y_2 \end{bmatrix}\text{,}\) so that the system weâ€™re trying to solve is
    \begin{align*} \amp \amp AX \amp = B \amp \amp \begin{bmatrix} 1 \amp 2 \\ 3 \amp 5 \end{bmatrix}\begin{bmatrix} x_1 \\ x_2 \end{bmatrix} = \begin{bmatrix} -1 \\ 2 \end{bmatrix} \\ \text{replace }A \text{ with } LU \amp \amp LUX \amp = B \amp \amp \begin{bmatrix} 1 \amp 0 \\ 3 \amp 1 \end{bmatrix}\begin{bmatrix} 1 \amp 2 \\ 0 \amp -1 \end{bmatrix}\begin{bmatrix} x_1 \\ x_2 \end{bmatrix} = \begin{bmatrix} -1 \\ 2 \end{bmatrix} \\ \text{replace }UX \text{ with } Y \amp \amp LY \amp = B \amp \amp \begin{bmatrix} 1 \amp 0 \\ 3 \amp 1 \end{bmatrix}\begin{bmatrix} y_1 \\ y_2 \end{bmatrix} = \begin{bmatrix} -1 \\ 2 \end{bmatrix} \end{align*}
    To see how this is easier to solve, look at the first row of \(LY=B\text{.}\) We see that \(y_1+0y_2=-1\text{,}\) which means that \(y_1=-1\text{.}\) The equation from the second row is
    \begin{equation*} 3y_1+y_2=-2\text{,} \end{equation*}
    and we can substitute the value of \(y_1\) we just found
    \begin{equation*} 3(-1)+y_2=-2 \end{equation*}
    and thus \(y_2=1\text{.}\)
    ðŸ”—
    We are trying to solve for \(X\) and we now know what \(Y\) equals, but remember that we defined \(UX=Y\text{,}\) which means that
    \begin{equation*} \begin{bmatrix} 1 \amp 2 \\ 0 \amp -1 \end{bmatrix}\begin{bmatrix}x_1 \\ x_2 \end{bmatrix}=\begin{bmatrix}-1 \\ 1 \end{bmatrix}\text{.} \end{equation*}
    Looking at the second row, we can see that \(x_2=-1\text{,}\) and then substituting into the equation from the first row gives us
    \begin{equation*} x_1+2x_2=-1\text{,} \end{equation*}
    so
    \begin{equation*} x_1+2(-1)=-1\text{,} \end{equation*}
    which means that \(x_1=1\text{.}\) Then the solution to \(AX=B\) is
    \begin{equation*} X=\begin{bmatrix} 1 \\ -1 \end{bmatrix}\text{.} \end{equation*}
    ðŸ”—
    ðŸ”—
    Why would we want to do this when we already have multiple other ways to solve a linear system? For small systems, we wouldnâ€™t. However, computers tend to use LU-decomposition when you ask them to solve matrix systems for the following reason. Suppose we had an \(1000\times 1000\) matrix \(A\) and needed to solve two systems \(AX=B_1\) and \(AX=B_2\text{.}\) Solving the first system would take about a billion operations, no matter whether we used Gauss-Jordan elimination or LU-decomposition. But if we used LU-decomposition and already had \(A=LU\text{,}\) then solving the second system would only take about a million more operations, instead of another billion to perform all of Gauss-Jordan elimination again on the second system. That is a significant time and memory savings, and also thereâ€™s orders of magnitude fewer opportunities for numerical errors to propagate.
    ðŸ”—
    Letâ€™s see another example for a larger matrix.

    Example 2.6.4.

    Find an \(LU\)-decomposition for
    \begin{equation*} A=\left[\begin{array}{rrr} 1 \amp 0 \amp 2 \\ 1 \amp 1 \amp -1 \\ -3 \amp -1 \amp -7 \end{array}\right]\text{.} \end{equation*}
    ðŸ”—
    Solution.
    The row operations we need to perform are
    ðŸ”—
    \(-R_1+R_2\rightarrow R_2\)
    ðŸ”—
    \(\left[\begin{array}{rrr} 1 \amp 0 \amp 2 \\ 0 \amp 1 \amp -3 \\ -3 \amp -1 \amp -7 \end{array}\right]\)
    ðŸ”—
    \(3R_1+R_3\rightarrow R_3\)
    ðŸ”—
    \(\left[\begin{array}{rrr} 1 \amp 0 \amp 2 \\ 0 \amp 1 \amp -3 \\ 0 \amp -1 \amp -1 \end{array}\right]\)
    ðŸ”—
    \(R_2+R_3\rightarrow R_3\)
    ðŸ”—
    \(\left[\begin{array}{rrr} 1 \amp 0 \amp 2 \\ 0 \amp 1 \amp -3 \\ 0 \amp 0 \amp -4 \end{array}\right]\)
    ðŸ”—
    We could perform one more row operation to scale the third row and put a 1 in the last position, but this is already an upper triangular matrix so we can just stop here and then
    \begin{equation*} U=\left[\begin{array}{rrr} 1 \amp 0 \amp 2 \\ 0 \amp 1 \amp -3 \\ 0 \amp 0 \amp -4 \end{array}\right]\text{.} \end{equation*}
    Then using elementary matrices to perform the row operations would result in
    \begin{equation*} E_3E_2E_1 A = U\text{,} \end{equation*}
    where \(E_1, E_2\text{,}\) and \(E_3\) are the elementary matrices
    \begin{align*} E_1=\left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ -1 \amp 1 \amp 0 \\ 0 \amp 0 \amp 1 \end{array}\right] \amp \amp E_2=\left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ 0 \amp 1 \amp 0 \\ 3 \amp 0 \amp 1 \end{array}\right] \amp \amp E_3=\left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ 0 \amp 1 \amp 0 \\ 0 \amp 1 \amp 1 \end{array}\right] \text{.} \end{align*}
    This means that
    \begin{align*} L \amp = E_1^{-1}E_2^{-1}E_3^{-1} \\ \amp = \left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ 1 \amp 1 \amp 0 \\ 0 \amp 0 \amp 1 \end{array}\right]\left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ 0 \amp 1 \amp 0 \\ -3 \amp 0 \amp 1 \end{array}\right] \left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ 0 \amp 1 \amp 0 \\ 0 \amp -1 \amp 1 \end{array}\right] \\ \amp = \left[\begin{array}{rrr} 1 \amp 0 \amp 0 \\ 1 \amp 1 \amp 0 \\ -3 \amp -1 \amp 1 \end{array}\right]\text{.} \end{align*}
    We can use Sage to verify our computations.
    ðŸ”—
    ðŸ”—
    ðŸ”—
    Letâ€™s put it all together and summarize the process.
    ðŸ”—

    Solving systems with LU-decomposition.

    If a square matrix doesnâ€™t require any interchanging of rows to put it into reduced row echelon form, then
    • Use elementary matrices \(E_1, E_2, \ldots E_k\) to perform the forward steps of Gauss-Jordan elimination. This means youâ€™ll have something of the form \(E_k\cdots E_2E_1A=U\text{,}\) where \(U\) is upper triangular.
      ðŸ”—
      ðŸ”—
    • Elementary matrices which perform the forward steps of Gauss-Jordan elimination (scale or multiply-and-add) are all lower-triangular, and so are their inverses. The product of lower triangular matrices is still lower triangular.
      ðŸ”—
      ðŸ”—
    • Multiply by the inverses in reverse order to obtain
      \begin{align*} A \amp =E_1^{-1}E_2^{-1}\cdots E_k^{-1}U\\ \amp = LU \text{,} \end{align*}
      where \(L\) is a lower triangular matrix.
      ðŸ”—
      ðŸ”—
    • To solve \(AX=B\text{,}\) we replace \(A\) with \(LU\) and then \(UX\) with \(Y\text{.}\) We can then solve the simpler systems \(LY=B\) and \(UX=Y\) with substitution.
      ðŸ”—
      ðŸ”—
    ðŸ”—
    ðŸ”—
    We will practice the method more and explore what happens if we do need to swap rows during the Participate material.

    Summary.

    • Computers are fast and can store lots of information, but they are prone to numerical errors due to binary representations and the use of finitely many digits. We can mitigate the effect of numerical errors using partial pivoting and by reducing the number of operations needed where errors could spread.
      ðŸ”—
      ðŸ”—
    • Every square matrix which doesnâ€™t need any row swap operations to perform Gauss-Jordan elimination can be expressed as the product of a lower triangular matrix with an upper triangular matrix.
